基于OpenZeppelinn建立标准代币

发表于 2018-02-16 | 分类于 Blockchain

本篇博文参考 Truffle 官网的一篇 Tutorial – BUILDING ROBUST SMART CONTRACTS WITH OPENZEPPELIN，简化的创建一种基于以太坊智能合约的代币。

准备工作

安装 truffle

Solidity 是开发 Ethereum 智能合约的语言，其语法与 JavaScript 相似。Solidity 是静态类型的，支持继承，库以及复杂的用户定义类型。其编译后就是节点运行的 EVM（Ethereum Virtual Machine）程序集。

Truffle 是当前最活跃的 Ethereum 开发框架，具有调试、部署和测试智能合约等功能。此次建立加密代币的教程就是基于 Truffle 框架的开发，首先开发机上要先安装好 Node.js ，默认带有 npm 软件包管理系统。

打开命令行工具安装 Truffle：

本博文使用的 Truffle 版本：

安装 ganache

在开发阶段需要测试，ganache 可以模拟智能合约所需要的以太坊区块链测试环境。

用官方的语言描述：

Quickly fire up a personal Ethereum blockchain which you can use to run tests, execute commands, and inspect state while controlling how the chain operates.

ganache 有桌面客户端以及命令行工具，本篇教程使用的是命令行工具搭配 Chrome 的 MetaMask 来进行测试。

认识 OpenZeppelin

建立的代币若要能通过以太币钱包收发，必须支持 ERC-20 标准。本教程使用 OpenZeppelin 库来简化建立加密代币的过程。

什么是 OpenZeppelin

OpenZeppelin是一套協助撰寫安全的加密合約的函式庫，裡面也提供了相容ERC20標準的智能合約。

解读
  • 首先引入 OpenZeppelin 库的 StandardToken.sol 文件，定义我们的 TutorialToken 智能合约，继承 StandardToken 智能合约。这样我们定义的 TutorialToken 智能合约就自动拥有 StandardToken 智能合约的属性及方法，我们也可以在我们定的智能合约里重写这些属性及方法；
  • 设置我们要发行的代币的属性，包括名字（今年狗年，我们就定义一个狗狗币吧）、符号（比特币的符号是BTC，这里我们定义为TT吧）、精度（最小交易单位，这里我们设置为2，即最小的交易额为0.01DT）、总的发行量（这里我们设置为10000）；

这样，我们自己的代币就创建成功了。当然，这只是一个简化的程序，线上的代币都是与金钱挂钩的，会面临巨大的攻击压力，还需要开发非常多的功能及做好健壮的安全工作，这就不是一篇小小的入门教程可以描述清楚的。

结语

智能合约一旦部署就难以修改，因此合约的安全性及其重要。虽然我们可以借助 OPENZEPPELIN 库创建相对来说比较安全的代币，并且对于开发者来说要避免重复造轮子，但是别人的轮子也不一定是完美的，所以不要过分依赖。在使用的同时要学习其中的思想，并且尝试去优化它。
